Thomas Heim // UX UI Designer

Squiggle Magic

For this project we were tasked to create the special content of the main game, to be delivered monthly. However, aside from the main loop of the game (which is to make the shapes shown in screen using the physical pieces available) we were given full freedom on how we wanted to deliver the experience of the game.

Initially the game was controlled by buttons on the bottom of the screen for each "room" in the level, however after user testing i realized that kids were intuitively dragging the screen to try and move the characters, so a new iteration was made taking that into account, i however left the buttons for the kids that preferred to do so.

The design of the ui was made with the main user in mind, as such we tried to give it a more rough, and hand drawn feel, with more literal icons.

In order to better fit with the game world, we made it so each level has its particular color theme, affecting the ingame UI as well as the activity screen.

Wireframe exploration

Final design

Customization was one of the new features considered for this content, as such a new screen, along with new flows had to be designed from the ground up.

Since the UI had to be on the simpler side on behalf of our target user, most of the work had to be done on designing the flows of the game, main example of this was designing the end of room flow, which had multiple triggers and would usually confuse users. The end product managed to organize all triggers in a narrative fashion in order to properly convey all the information needed.
